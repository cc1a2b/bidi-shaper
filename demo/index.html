Browsers and HarfBuzz run two invisible passes before any glyph hits the screen: the bidirectional algorithm and Arabic shaping. Renderers that place glyphs one after another run neither. bidi-shaper runs both and hands you a plain string.

٠١ · the errata

What a naive renderer gets wrong

Give a glyph-by-glyph renderer logical-order Arabic and four things break at once. bidi-shaper fixes all four before the first glyph is drawn.

The textDrawn naively, left-to-rightThrough bidi-shaper

Reading direction runs come out reversed, end-first RTL runs flow right-to-left, LTR stays LTR
Numbers in RTL drift to the wrong end of the line stay left-to-right, in place — سنة ١٤٤٧

٠٢ · the galley

Set your own line

Type anything — Arabic, Persian, Urdu, mixed with English and numbers. Both canvases place each code point left-to-right, exactly like a renderer that can't shape or reorder. Only the second one is fed bidi-shaper's output.

The compositor's bench المنضدة

One call — render(text, options) — shapes, reorders, and mirrors. Copy the output straight into doc.text(...).

The output string is in visual order: its code points are already shaped presentation forms in drawing sequence. Hand it to any renderer that lays glyphs out left-to-right.

Embedding levels المستويات

analyze(text) resolves a level per code point. Even is left-to-right, odd is right-to-left — the spine of the bidi algorithm.

even = LTR odd = RTL

Contextual shaping التشكيل

One Arabic letter is four glyphs, chosen by its neighbours. Pick a letter to see the forms shape() substitutes.

Drop-in for your renderer المحوّلات

Structurally-typed adapters — they never import the host library, so they add nothing to your bundle. Same engine underneath.

jsPDF bidi-shaper/jspdf
import { rtlText } from "bidi-shaper/jspdf";

doc.text(rtlText("مرحبا بالعالم"), 20, 40);
PDFKit bidi-shaper/pdfkit
import { textBidi } from "bidi-shaper/pdfkit";

// shapes + reorders; features:[] so fontkit won't re-shape
textBidi(doc, "سلام دنیا", 72, 80);
Canvas 2D bidi-shaper/canvas
import { fillTextBidi } from "bidi-shaper/canvas";

fillTextBidi(ctx, "قائمة (أ)", x, y);
three.js bidi-shaper/three
import { prepareText } from "bidi-shaper/three";

const label = prepareText("اردو میں خوش آمدید");
// → feed to TextGeometry / troika-text
