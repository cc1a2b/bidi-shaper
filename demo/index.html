٠٢ · the errata

Four things break at once

Give a glyph-by-glyph renderer logical-order Arabic and it gets every one of these wrong. bidi-shaper fixes all of them before the first glyph is drawn.

٠٣ · the bench

Set your own line

Everything below is computed live by the library source in your browser — no precomputed strings. Type anything; the canvases place each code point left-to-right, exactly like a renderer that can't shape or reorder.

The compositor's bench المنضدة

One call — render(text, options) — shapes, reorders, and mirrors. Copy the output straight into doc.text(...).

your string, glyph-by-glyph broken
render() output, same renderer set

The output is in visual order: its code points are already shaped presentation forms in drawing sequence. Hand it to any renderer that lays glyphs out left-to-right.

Embedding levels المستويات

analyze(text) resolves a level per code point. Even is left-to-right, odd is right-to-left — the spine of the algorithm, and the colours on the loom.

even = LTR odd = RTL

Drop-in for your renderer المحوّلات

Structurally-typed adapters — they never import the host library, so they add nothing to your bundle. Same engine underneath.

jsPDF bidi-shaper/jspdf
import { rtlText } from "bidi-shaper/jspdf";

doc.text(rtlText("مرحبا بالعالم"), 20, 40);
PDFKit bidi-shaper/pdfkit
import { textBidi } from "bidi-shaper/pdfkit";

// shapes + reorders; features:[] so fontkit won't re-shape
textBidi(doc, "سلام دنیا", 72, 80);
Canvas 2D bidi-shaper/canvas
import { fillTextBidi } from "bidi-shaper/canvas";

fillTextBidi(ctx, "قائمة (أ)", x, y);
three.js bidi-shaper/three
import { prepareText } from "bidi-shaper/three";

const label = prepareText("اردو میں خوش آمدید");
